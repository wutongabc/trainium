## ASSISTANT (2026-07-20 09:35:25)

定位到了：不是 Opus 4.6、Region 或 AWS 权限问题，而是 Claude Code 使用了无效的 Bedrock bearer API key。

证据：

- Claude 日志明确返回：`403 Authentication failed: Please make sure your API Key is valid.`
- 当前实际配置目录是 `/dev3/cc`，因为设置了：
  `CLAUDE_CONFIG_DIR=/dev3/cc`
- 所以真正生效的是 [/dev3/cc/settings.json](/dev3/cc/settings.json)，不是默认的 `~/.claude/settings.json`。
- 生效配置里的 `AWS_BEARER_TOKEN_BEDROCK` 与当前 shell 环境中的同名变量内容不同。
- 我使用当前 AWS IAM 登录凭证直接调用：
  `us.anthropic.claude-opus-4-6-v1`
  已成功返回 `OK`。
- 因此账号、`us-east-2`、Opus 4.6 跨区域 inference profile 和模型权限均正常。

推荐修复：使用已有的 AWS IAM 凭证链，删除 bearer token 配置：

```json
{
  "env": {
    "CLAUDE_CODE_USE_BEDROCK": "1",
    "AWS_REGION": "us-east-2",
    "ANTHROPIC_DEFAULT_OPUS_MODEL": "us.anthropic.claude-opus-4-6-v1",
    "ANTHROPIC_DEFAULT_SONNET_MODEL": "us.anthropic.claude-sonnet-4-5-20250929-v1:0",
    "ANTHROPIC_DEFAULT_HAIKU_MODEL": "us.anthropic.claude-haiku-4-5-20251001-v1:0"
  }
}
```

也就是从 `/dev3/cc/settings.json` 删除：

```json
"AWS_BEARER_TOKEN_BEDROCK": "..."
```

另外建议清理 shell 中这两个变量，避免来源混乱：

```bash
unset AWS_BEARER_TOKEN_BEDROCK
unset AWS_KEY
```

`AWS_KEY` 不是 Claude Code/Bedrock 的标准认证变量，不应使用。

然后重新启动：

```bash
claude --model us.anthropic.claude-opus-4-6-v1
```

如果你必须使用 Bedrock API key，而不是 IAM 登录凭证，那么需要在 AWS 控制台重新生成有效 key，并确保只在 `/dev3/cc/settings.json` 或 shell 二选一配置，避免现在这种两个不同 key 同时存在的情况。
