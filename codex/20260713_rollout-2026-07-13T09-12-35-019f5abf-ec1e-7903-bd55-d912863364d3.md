## ASSISTANT (2026-07-13 09:16:13)

最推荐用 `rsync`，并让“源服务器 A”直接连接“目标服务器 B”，这样本地电脑不承载 300G 数据。

假设：

- 源服务器：`A`
- 目标服务器：`B`
- 源目录：`/data/source/`
- 目标目录：`/data/backup/`

先从本地登录 A：

```bash

tmux new -s transfer
```

在 A 上测试能否连接 B：

```bash
ssh userB@B 'hostname'
```

如果成功，执行传输：

```bash
rsync -aH --info=progress2 --partial --append-verify \
  /data/source/ userB@B:/data/backup/
```

注意源路径最后的 `/`：表示复制目录内容。传输中断后，重新执行同一条命令即可继续。退出 `tmux` 但保持任务运行：

```text
Ctrl-B，然后按 D
```

之后可重新查看：

```bash
tmux attach -t transfer
```

如果 A 无法 SSH 到 B，需要先在 A 上配置密钥：

```bash
ssh-keygen -t ed25519
ssh-copy-id userB@B
```

如果两台服务器彼此完全不通，只能通过本地中转，临时可以用：

```bash
scp -3 -r userA@A:/data/source userB@B:/data/backup/
```

但这种方式会占用本地上下行带宽，而且断线后不方便续传。300G 数据最好使用上面的服务器直连方式。传输前也建议确认目标磁盘空间：


## ASSISTANT (2026-07-13 09:35:44)

你之前记得的一行大概率是：

```bash
sudo mount /dev/nvme1n1 /dev3
```

但这条命令只负责“挂载已有文件系统”，不会创建文件系统。你这块盘是裸盘，所以会报：

```text
wrong fs type
```

磁盘初始化过程是：

```text
裸磁盘 → 分区 → 文件系统 → 挂载目录
```

各命令含义：

- `mklabel gpt`：创建 GPT 分区表，4.9TB 磁盘适合使用 GPT。
- `mkpart`：创建一个分区 `/dev/nvme1n1p1`。
- `partprobe`：让系统重新读取分区表。
- `mkfs.ext4`：创建 ext4 文件系统，会清除该分区数据。
- `mount`：把文件系统挂载到 `/dev3`。
- `chown`：让 `ubuntu` 用户可以写入。

如果确认是空盘，可以合并成一行：

```bash
sudo parted -s /dev/nvme1n1 mklabel gpt && sudo parted -s -a optimal /dev/nvme1n1 mkpart primary ext4 0% 100% && sudo partprobe /dev/nvme1n1 && sudo mkfs.ext4 -L transfer /dev/nvme1n1p1 && sudo mkdir -p /dev3 && sudo mount /dev/nvme1n1p1 /dev3 && sudo chown ubuntu:ubuntu /dev3
```

不过建议分开执行，哪一步失败会更容易排查。
