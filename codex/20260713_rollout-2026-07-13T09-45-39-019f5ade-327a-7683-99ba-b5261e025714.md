## ASSISTANT (2026-07-13 09:48:55)

如果你指的是 issue 里的 `--enable-experimental-O1`：**有机会明显缩短编译时间，但不保证单个 compile 全核跑满**。

这个功能本来就是通过并行执行部分 compiler passes 来降低编译时间；旧版默认使用 8 个进程。但 Neuron 官方也说明，有些 pass 仍然是串行的，所以“一核忙、多核闲”可能在某些阶段继续出现。[Issue #646](https://github.com/aws-neuron/aws-neuron-sdk/issues/646)

现代 SDK 已经弃用旧参数，改用：

```bash
export NEURON_CC_FLAGS="${NEURON_CC_FLAGS:-} --optlevel 1"
python train.py ...
```

也就是 `--optlevel 1` / `-O1`。它优先缩短编译时间；默认是 `O2`，`O3` 编译更慢但可能获得更好的运行性能。[Neuron 编译器 release notes](https://awsdocs-neuron.readthedocs-hosted.com/en/latest/release-notes/components/compiler.html)

需要注意：

- O1 适合研发迭代，最终模型要重新比较吞吐/延迟。
- 换成 O1 会导致第一次重新编译；之后可由 persistent cache 复用。
- 如果 PyTorch/XLA 会产生多个 HLO graph，`neuron_parallel_compile` 通常更有效：它会提取多个 graph，然后启动多个 compiler process 并行编译。[官方文档](https://awsdocs-neuron.readthedocs-hosted.com/en/latest/frameworks/torch/torch-neuronx/api-reference-guide/training/pytorch-neuron-parallel-compile.html)

```bash
neuron_parallel_compile torchrun --nproc_per_node=N train.py --steps_this_run=10
# 之后正常运行原命令
```

结论：先试 `--optlevel 1`；如果是多 graph 的 PyTorch/XLA training，再配合 `neuron_parallel_compile`。不要期待它把一个巨大的单 graph 变成 100% 多核并行。

## ASSISTANT (2026-07-13 09:59:04)

有，**这个开发版值得利用**，但它解决的是“多个独立 HLO 编译任务并行”，不是把一个巨大的 `neuronx-cc` 任务拆成多核。

我看到的关键点：

- 新增 `CompilationThreadPool`，默认 16 个前端 worker，可用 `NEURON_PARALLEL_COMPILE_WORKERS` 调整。[CompilationThreadPool.h](/dev3/zigeng/torch-neuronx-develop/torch_neuronx/csrc/core/compilation/CompilationThreadPool.h:13)
- 默认启动编译服务，由多个 worker 分别运行多个 `neuronx-cc` 子进程；实际编译进程数由 `NEURON_COMPILE_SERVICE_WORKERS` 控制。[NeuronCompilationService.cpp](/dev3/zigeng/torch-neuronx-develop/torch_neuronx/csrc/core/compilation/service/NeuronCompilationService.cpp:22)
- 编译服务启用时走异步并行路径；禁用或启动失败后，会退回单线程直接编译。[OperationExecutionEngine.cpp](/dev3/zigeng/torch-neuronx-develop/torch_neuronx/csrc/core/OperationExecutionEngine.cpp:221)
- 这个版本的 XLA 路径默认是 `-O1`，但 `torch.compile` 路径默认仍是 `-O2`；可用 `NEURON_COMPILER_OPT_LEVEL` 覆盖。[KernelExecution.cpp](/dev3/zigeng/torch-neuronx-develop/torch_neuronx/csrc/core/KernelExecution.cpp:44)

所以：

| 你的场景 | 预期收益 |
|---|---|
| 多个 HLO、多个 rank/stream | 比较可能明显加速 |
| 单个巨大 HLO | 主要仍靠 `-O1`，不会保证多核 |
| 直接手动运行 `neuronx-cc compile` | 这个 repo 基本帮不上忙 |

运行时可以先尝试：

```bash
NEURON_COMPILE_SERVICE=1 \
NEURON_COMPILE_SERVICE_WORKERS=2 \
NEURON_PARALLEL_COMPILE_WORKERS=4 \
NEURON_COMPILER_OPT_LEVEL=-O1 \
<原来的启动命令>
```

不要一开始把 worker 设到 16 或 64；多个 `neuronx-cc` 进程会同时吃 CPU 和内存，容易反而 OOM 或变慢。
