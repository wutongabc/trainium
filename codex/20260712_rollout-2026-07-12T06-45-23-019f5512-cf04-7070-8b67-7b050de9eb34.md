# FIFO Experiment Queue for AWS Trn2/Neuron

## Summary

Add a persistent single-worker CLI that accepts Bash experiment scripts, stores them in FIFO order, runs one full-device experiment at a time, and preserves logs/status across terminal or tmux disconnects.

No accelerator allocation logic will be injected: each submitted script remains responsible for its own `TP_SIZE`, model server, ports, and Neuron configuration.

## Key Changes

- Add `scripts/experiment_queue.py` using only the Python standard library.
- Add SQLite-backed state under `.experiment_queue/`, with generated logs and daemon metadata excluded via `.gitignore`.
- Support commands:

  ```bash
  python scripts/experiment_queue.py start
  python scripts/experiment_queue.py submit --name tongyi-get \
    --cwd . --env MAX_QUERIES=1 -- \
    bash/neuron/run_tongyi_neuron.bash
  python scripts/experiment_queue.py list
  python scripts/experiment_queue.py status JOB_ID
  python scripts/experiment_queue.py logs JOB_ID --follow
  python scripts/experiment_queue.py cancel JOB_ID
  python scripts/experiment_queue.py pause
  python scripts/experiment_queue.py resume
  python scripts/experiment_queue.py stop
  ```

- Submission records the absolute script path, positional arguments, working directory, explicit environment overrides, name, submission time, and job ID. Commands are stored as argument arrays rather than shell strings.
- The worker:
  - claims jobs atomically in ascending job-ID order;
  - runs each script as `/bin/bash` in its recorded working directory;
  - starts each job in a separate Unix process group;
  - writes combined stdout/stderr to a per-job log;
  - records start/end times and exit status;
  - marks failures and continues to the next queued job;
  - performs no automatic retries.
- `cancel` handles both queued and running jobs. Running jobs receive graceful termination through their process group, followed by forced termination after a timeout if needed, ensuring child Neuron/vLLM processes are cleaned up.
- `pause` prevents new jobs from starting while allowing the current job to finish. `stop` terminates the active job and worker while leaving remaining queued jobs intact. Restarting the worker detects stale running jobs, marks them interrupted, cleans their recorded process group when present, and continues with queued jobs.
- `start` launches a detached worker suitable for use from tmux and refuses duplicate workers through a lock/PID mechanism.
- `list` displays queued/running/completed/failed/cancelled/interrupted jobs, with compact timestamps and exit codes.

## Test Plan

Add focused tests covering:

- FIFO ordering with multiple submitted jobs.
- Persistence and reload of job metadata through SQLite.
- Propagation of working directory, arguments, and explicit environment variables.
- Failure recording while later jobs continue.
- Combined stdout/stderr log creation and `--follow` behavior.
- Queued-job cancellation.
- Running-job cancellation including child-process cleanup.
- Pause/resume behavior.
- Duplicate worker prevention.
- Recovery of stale running jobs after worker interruption.
- CLI smoke tests using temporary Bash scripts; no real Neuron hardware required.

Document usage and operational assumptions in `docs/experiment_queue.md`, including the Trn2 workflow, log locations, cancellation behavior, and examples for existing scripts such as `bash/neuron/run_tongyi_neuron.bash`.
</proposed_plan>

# htop 风格的实验队列终端界面

## Summary

将程序改为全屏终端 TUI，默认启动后直接进入界面：

```bash
./expq
```

同时保留非交互式 CLI，方便批量提交：

```bash
./expq submit --name test -- bash/neuron/run_tongyi_neuron.bash
```

`python scripts/experiment_queue.py` 仍然可以作为备用启动方式。

后台 worker 与 TUI 分离。关闭 TUI 不会停止正在运行的实验。

## TUI 功能

界面类似 `htop`，实时显示：

- worker 状态：运行、暂停、停止
- 当前实验及运行时间
- FIFO 队列中的任务
- 成功、失败、取消和中断任务
- exit code
- 当前实验日志摘要
- Trn2/Neuron 资源状态（如果 `neuron-ls` 可用）

主要按键：

- `↑/↓` 或 `j/k`：选择任务
- `a`：添加实验
- `Enter`：查看任务详情
- `l`：查看并滚动日志
- `f`：跟随实时日志
- `c`：取消选中的任务
- `p`：暂停队列
- `r`：恢复队列
- `s`：启动或重启 worker
- `x`：停止 worker
- `q`：退出 TUI，但不停止后台实验

添加实验时提供表单：

1. Bash 脚本路径
2. 脚本参数，使用 shell 风格解析
3. 工作目录，默认当前目录
4. 环境变量，每行一个 `KEY=VALUE`

## 后台调度

- SQLite 持久化任务状态。
- 所有任务按提交顺序 FIFO 执行。
- 同时只运行一个完整设备实验。
- 每个任务使用独立 Unix process group。
- 取消任务时终止 Bash 脚本及其子进程，例如 vLLM/Neuron server。
- 任务失败后记录 exit code，并继续下一个任务。
- 不自动重试。
- worker 重启后识别并清理上次遗留的运行任务，标记为 `interrupted`。
- 日志保存到 `.experiment_queue/logs/<job_id>.log`。
- `.experiment_queue/` 加入 `.gitignore`。

队列不会自动修改 `TP_SIZE`、端口或 Neuron 配置；实验脚本仍负责声明自己使用全部 tensor core。

## 文件和接口

- `scripts/experiment_queue.py`：核心 CLI、worker、TUI。
- `expq`：无需输入 Python 命令的启动包装器。
- `tests/test_experiment_queue.py`：队列、worker、取消、恢复和 CLI 测试。
- `docs/experiment_queue.md`：使用说明、按键说明和 Trn2 示例。

保留 CLI：

```bash
./expq start
./expq submit ...
./expq list
./expq status JOB_ID
./expq logs JOB_ID --follow
./expq cancel JOB_ID
./expq pause
./expq resume
./expq stop
```

无参数运行 `./expq` 时进入 TUI。

## 测试计划

- FIFO 顺序测试。
- TUI 添加任务和参数解析测试。
- 环境变量、工作目录和脚本参数传递测试。
- 失败任务继续执行测试。
- 日志显示和实时跟随测试。
- 运行中任务及其子进程取消测试。
- worker 重启恢复测试。
- 重复启动 worker 防护测试。
- CLI 与 TUI 共用状态数据库测试。
- 在 tmux 中进行一次人工 smoke test，确认关闭界面后任务仍继续运行。
