## Context

We need to implement a single `Qwen3ForCausalLM` model class that supports both:
1. **Qwen3-32B** - Dense model (no MoE), 32 layers, GQA (num_attention_heads=32, num_key_value_heads=8)
2. **Tongyi-DeepResearch-30B-A3B** - MoE model, 128 experts, top-8 routing, 48 layers

Both models share:
- No attention bias (`attention_bias: False`)
- YaRN RoPE with theta=10000.0
- RMSNorm (eps=1e-6)
- Same attention architecture (just different layer counts)

## Architecture from Exploration

### GptOss Attention (to adapt from)
**File**: `/dev3/zigeng/bc/vllm-neuron/vllm_neuron/model/gpt_oss/model_bf16.py` (Lines 279-968)

**Key components**:
- XLA kernels: `NF.segmented_attention()`, `NF.flash_attention()`, `NF.attention_decode()`
- RoPE: `GptOssRotaryEmbedding` class with YaRN scaling
- TP sharding: QKV fused with attention DP, O projection with all-reduce
- Weight loaders: `fused_qkv_weight_loader()`, `o_proj_weight_loader()` from `weight_loaders_bf16.py`
- Bias: Already conditionally supported (`if config.attention_bias`)

**Changes needed**:
- Remove sinks (learnable attention sinks, Qwen3 doesn't have them)
- Remove sliding window (Qwen3 doesn't use it)
- Verify RoPE parameters match Qwen3 (should be compatible)

### GptOss MoE (to adapt from)
**File**: `/dev3/zigeng/bc/vllm-neuron/vllm_neuron/model/gpt_oss/model_bf16.py` (Lines 976-1450)

**Key components**:
- Router: Replicated weights `[E_total, H]`, top-k with `torch.topk()` + softmax
- Experts: Fused `gate_up_proj_weight [E_local, H, I*2]`, `down_proj_weight [E_local, I, H]`
- TP/EP sharding: Linear expert placement, intermediate size sharded for TP
- Kernels: `NF.moe_cte()` (prefill), `NF.moe_block_tkg()` (decode)
- Weight loaders: `expert_gate_up_weight_sharding_loader()`, `expert_down_weight_sharding_loader()`

**Changes needed**:
- Adapt router for Qwen3Moe's architecture (check if normalization differs)
- Adjust expert count (128 vs 8) and top-k (8 vs 4)
- Verify activation function (Qwen3 uses SiLU, GptOss uses Swish with alpha=1.702)
